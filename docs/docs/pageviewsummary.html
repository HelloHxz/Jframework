核心概述Summary
该框架虽然为Web框架,但是通过框架的封装已经无需再使用HTML代码
而是通过JSON声明的方式进行开发
设计思路
1. UI文件
组件由JSON配置声明
复杂组件由多个组件组成
多个组件配置形成一个页面配置
多个页面在路由的配合下则形成一个APP应用。
存放在 ～./Apps/[具体项目名称]/pages/
2. 逻辑文件
一个PageView UI配置文件对应一个逻辑处理文件
该逻辑处理文件对象实例根据页面配置进行组件实例化
在该逻辑文件中进行页面生命周期事件,以及组件生命周期事件以及交互事件的代码编写
该逻辑对象实例同事也对页面,组件,组件之间的状态,变量以及数据进行维护,成为当前页面及所有页面组件的上下文
存放在 ～./Apps/[具体项目名称]/logic/
